# Contributing to FlowGrid

Thank you for your interest in improving FlowGrid! This document explains how to get started and the expectations for contributions.

1. Fork the repository and create a feature branch from `main`.
2. Keep changes small and focused. Add tests where applicable.
3. Run linters and unit tests locally before creating a PR.
4. Open a pull request with a clear description of the change and link to any related issues.

Code style
- Use the existing code style for both client and server. If ESLint/Prettier configurations exist, use them.

Reporting bugs
- Open an issue with clear reproduction steps, expected vs actual behavior, and environment details.

Thank you — maintainers will review PRs and respond as soon as possible.
